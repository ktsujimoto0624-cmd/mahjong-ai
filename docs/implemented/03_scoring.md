# 03. 点数計算（役判定・符計算・ドラ）

## 概要
和了時の役判定、符計算、翻→点数変換、ドラ加算を行う。

## 処理フロー

```
和了 → 手牌分解(全パターン) → 各パターンで役判定
     → 最高翻のパターンを選択 → 符計算 → 基本点算出 → 支払い計算
     → ドラ加算(game.py側)
```

## 役判定 (`yaku.py`)

### 入力: win_info
```python
{
    "hand": カウント配列（14枚）,
    "melds": 副露リスト,
    "winning_tile": 和了牌ID,
    "is_tsumo": ツモ和了か,
    "is_riichi": リーチしていたか,
    "seat_wind": 自風(0=東,1=南,2=西,3=北),
    "round_wind": 場風(0=東,1=南,2=西,3=北),
    "is_menzen": 門前かどうか,
}
```

### 最高得点選択
手牌の分解は複数パターンありうる。全パターンで役判定し、翻数が最大のパターンを採用する。

## 符計算 (`score.py`)

### 符の構成

| 要素 | 符 |
|------|-----|
| **副底** | 門前ロン: 30符 / ツモ or 鳴き: 20符 |
| **面子** | 下表参照 |
| **雀頭** | 役牌(三元牌/場風/自風): 2符 |
| **待ち** | 嵌張/辺張/単騎: 2符 / 両面: 0符 |
| **ツモ** | 2符（平和ツモは除く） |

### 面子の符

| | 中張牌 | 么九牌/字牌 |
|---|--------|------------|
| 明順子 | 0 | 0 |
| 明刻子 | 2 | 4 |
| 暗刻子 | 4 | 8 |
| 明槓子 | 8 | 16 |
| 暗槓子 | 16 | 32 |

### 特殊ケース
- **七対子**: 固定25符
- **平和ツモ**: 固定20符
- 10の位に切り上げ

## 翻→点数変換

### 基本点
| 翻数 | 基本点 |
|------|--------|
| 1-4翻 | 符 × 2^(翻+2) (上限2000) |
| 5翻 | 2000 (満貫) |
| 6-7翻 | 3000 (跳満) |
| 8-10翻 | 4000 (倍満) |
| 11-12翻 | 6000 (三倍満) |
| 13翻以上 | 8000 (数え役満/役満) |

### 支払い
- **子ロン**: 基本点 × 4
- **親ロン**: 基本点 × 6
- **子ツモ**: 親から基本点×2、子2人から基本点×1
- **親ツモ**: 子3人から基本点×2
- 100点単位に切り上げ

## ドラ (`game.py` 内で処理)

- ドラ表示牌からドラ牌を特定（`tile.py: dora_from_indicator`）
- 手牌+副露に含まれるドラ枚数をカウント
- 役ありの場合のみ、ドラ翻を加算
- カンのたびにドラ表示牌が1枚追加される
