# 01. エンジンコア（牌・山・プレイヤー）

## 概要
麻雀の基本データ構造と、1局のゲーム進行を管理するコア部分。

## 牌の内部表現 (`tile.py`)

牌は0〜33の整数IDで表現する（カウント方式）。

| 範囲 | 牌種 |
|------|------|
| 0-8 | 萬子 (1m-9m) |
| 9-17 | 筒子 (1p-9p) |
| 18-26 | 索子 (1s-9s) |
| 27-33 | 字牌 (東南西北白發中) |

手牌は34要素のリスト（カウント配列）で管理する。`hand[0]=2` は1萬を2枚所持。

### 主要関数
- `tile_name(id)` — 表示名（例: "1m", "東"）
- `is_suit(id)` / `is_honor(id)` — 数牌/字牌判定
- `is_terminal(id)` — 老頭牌(1,9)判定
- `dora_from_indicator(id)` — ドラ表示牌→ドラ牌変換

## 山 (`wall.py`)

136枚の牌をシャッフルし、順にツモる。

- **王牌**: 末尾14枚（嶺上牌4枚 + ドラ表示5枚 + 裏ドラ表示5枚）
- `draw()` — 山から1枚ツモ（王牌に到達するとNone）
- `draw_from_dead_wall()` — 嶺上牌をツモ（カン時）
- `get_dora_indicators()` — 現在のドラ表示牌
- `add_dora_indicator()` — カンドラ追加

## プレイヤー (`player.py`)

1人のプレイヤーの状態を管理する。

### 保持するデータ
- `hand` — 手牌（カウント配列）
- `discards` — 河（捨て牌履歴）
- `melds` — 副露リスト
- `points` — 持ち点
- `is_riichi` — リーチ状態
- `riichi_turn` — リーチ宣言巡目

### 副露メソッド
- `add_chi(taken, hand_tiles, from_seat)` — チー
- `add_pon(taken, from_seat)` — ポン
- `add_daiminkan(taken, from_seat)` — 大明槓
- `add_ankan(tile_id)` — 暗槓
- `add_kakan(tile_id)` — 加槓

## 和了判定 (`agari.py`)

### 和了形の3パターン
1. **基本形**: 4面子 + 1雀頭
2. **七対子**: 7種×2枚
3. **国士無双**: 么九牌13種 + 1枚重複

### 主要関数
- `is_agari(hand, melds_count)` — 和了形かどうか判定
- `shanten_number(hand, melds_count)` — シャンテン数計算（-1=和了, 0=テンパイ）
- `decompose_regular(hand, melds_count)` — 面子+雀頭の全分解パターン列挙

## ゲームループ (`game.py`, `game_naki.py`)

1局の進行を管理する `GameRound` クラス。

### フロー
```
配牌(13枚×4人)
  ↓
ツモ → ツモ和了判定 → 暗槓/加槓判定 → リーチ判定 → 打牌
  → ロン判定(頭ハネ) → ポン/大明槓判定 → チー判定
  → 次のプレイヤー → ツモ ... (山が尽きたら流局)
```

### 鳴きの優先順位
1. ロン（最優先）
2. ポン / 大明槓（全員対象）
3. チー（下家のみ）

### カン処理
カン成立 → 嶺上牌ツモ → 新ドラ表示 → 嶺上ツモ和了判定 → 打牌フェーズへ
