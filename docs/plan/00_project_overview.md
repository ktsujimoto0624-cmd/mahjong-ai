# 麻雀AI プロジェクト概要

## ビジョン
日本式リーチ麻雀（4人打ち）のAIエージェントを構築し、
自己対戦を通じて進化させることで、麻雀の真理に迫る。

## 基本方針
- **段階的アプローチ**: ルールベース → 機械学習（強化学習）へ段階的に進化
- **ライフワーク**: 長期的に取り組み、各段階を楽しみながら深める
- **理論重視**: 設計・理論を議論しながら実装を進める
- **既存研究活用**: Suphx等の先行研究、公開棋譜データを参考にする

## 開発環境
- OS: Windows 11
- GPU: なし（CPU前提、将来的にクラウドGPU検討）
- 言語: Python（ML/AIエコシステムが最も充実）
- 開発者背景: C++経験あり（30年前）、ML未経験

## ロードマップ

### Phase 1: 基盤構築
麻雀のルールエンジンと基本的な対戦シミュレータを作る。

- 麻雀の牌・手牌・山・河などのデータ構造定義
- ルール実装（ツモ・打牌・鳴き・リーチ・和了判定・点数計算）
- 4人対戦のゲームループ
- ランダムに打つだけのエージェント（動作確認用）

### Phase 2: ルールベースAI
確率・期待値に基づく戦略をコードで実装する。

- シャンテン数計算（和了までの最短距離）
- 受入枚数・有効牌の計算
- 期待値に基づく打牌選択
- 基本的な押し引き判断（危険牌推定）
- ルールベースエージェント同士の対戦・統計収集

### Phase 3: 機械学習への移行
棋譜データと強化学習を用いてAIを進化させる。

- 公開棋譜データの取得と前処理
- 局面の特徴量設計（何を入力とするか）
- 教師あり学習: 棋譜から「強い人の打ち方」を模倣
- 強化学習: 自己対戦による戦略の改善
- ルールベースAI vs 学習AIの比較評価

### Phase 4: 高度化・探求
より深い戦略の領域へ。

- 相手モデリング（相手の手牌・戦略の推定）
- 状況に応じた戦略切替（トップ目・ラス目での判断変化）
- 鳴き判断の最適化
- 長期戦略（半荘通しての順位最大化）

## 先行研究・参考資料

### AI研究
- **Suphx** (Microsoft Research, 2020): 天鳳で十段達成。深層強化学習ベース
- **NAGA**: 麻雀AI解析ツール。牌譜解析に利用可能
- **Mortal**: オープンソースの麻雀AI。強化学習ベース

### 棋譜データ
- **天鳳の牌譜**: 天鳳の対局ログ（公開されている範囲）
- **雀魂の牌譜**: 一部取得可能
- **Mortalプロジェクト**: 学習用データセットを公開

### ライブラリ・ツール
- **mahjong** (Python): 麻雀ユーティリティライブラリ
- **PyTorch**: 機械学習フレームワーク（将来利用）
- **OpenSpiel** (Google): ゲームAI研究フレームワーク（麻雀対応あり）
