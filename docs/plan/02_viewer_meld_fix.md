# 02. ビューア副露表示バグ修正計画

## 問題点（スクリーンショットより）

1. **副露が手牌と混在** — 鳴いた牌グループが手牌の中に混ざって表示されている
2. **副露の位置が不正** — 手牌の右側に分離して表示されるべき
3. **鳴いた牌の横向き表示がない** — 実際の麻雀では、誰から鳴いたか分かるように1枚を横向きにする
4. **河から鳴いた牌が正しく消えていない可能性**
5. **手牌枚数が副露分だけ減っていない可能性**

## 麻雀の副露表示ルール

### 横向き牌の位置（誰から鳴いたかを示す）
鳴いたプレイヤーから見た相対位置で、横向き牌の置き場所が決まる。

| 鳴き元 | 相対位置 | 横向き牌の位置 |
|--------|----------|---------------|
| 上家（左の人） | (from-self+4)%4 == 3 | 左端 |
| 対面（正面の人） | (from-self+4)%4 == 2 | 中央 |
| 下家（右の人） | (from-self+4)%4 == 1 | 右端 |

### 各鳴きの表示

**チー:**
- 横向き牌は常に左端（上家からしか鳴けないため）
- `[横][縦][縦]` — 順子を数字順に並べ、鳴いた牌だけ横向き

**ポン:**
- 横向き牌の位置は鳴き元による（上表参照）
- 例: 上家から → `[横][縦][縦]`
- 例: 対面から → `[縦][横][縦]`
- 例: 下家から → `[縦][縦][横]`

**大明槓:**
- ポンと同様に横向き牌の位置を決める（4枚）
- 横向き牌の上に1枚重ねる形

**加槓:**
- 元のポンの横向き牌の上に4枚目を重ねる

**暗槓:**
- 両端が裏向き、中央2枚が表向き
- `[裏][表][表][裏]`

### 副露の表示位置
- 手牌の**右側**に配置（プレイヤーの視点から）
- 手牌と副露の間にスペースを空ける
- 左右プレイヤーの場合は手牌の**下側**に配置

## 修正対象ファイル

| ファイル | 修正内容 |
|---------|---------|
| `viewer/game_viewer/scripts.py` | `renderMelds()` の描画ロジック全面書き直し |
| `viewer/game_viewer/styles.py` | 横向き牌のCSS、副露位置のレイアウト修正 |
| `viewer/game_viewer/generator.py` | 必要に応じてHTML構造調整 |

## 実装手順

### Step 1: JavaScript — renderMelds() の書き直し

**現在の問題:** 牌を順番に並べるだけで、横向き表示や位置計算がない。

**修正内容:**
1. `renderMelds(seat, isVertical)` に seat 番号を渡す（既に渡されている）
2. 鳴き元との相対位置を計算: `relPos = (from_seat - seat + 4) % 4`
3. 相対位置に応じて横向き牌の挿入位置を決定
4. チー: 牌を数字順にソートし、taken_tileの位置を左端に固定
5. ポン: relPos に応じて左/中/右に横向き牌を配置
6. 暗槓: 両端に裏向き表示用クラスを付与
7. 加槓: 横向き牌の上に4枚目を重ねる

**横向き牌のHTML:**
```javascript
// 横向き（水平プレイヤー）
'<img class="tile-meld tile-meld-sideways" src="...">'

// 横向き（垂直プレイヤー、左右）
'<img class="tile-meld-vert tile-meld-sideways" src="...">'
```

### Step 2: CSS — 横向き牌とレイアウト

**追加するクラス:**
```css
/* 横向き牌（鳴いた牌） */
.tile-meld-sideways {
    transform: rotate(90deg);
    /* マージン調整で隙間をなくす */
}

/* 暗槓の裏向き牌 */
.tile-meld-facedown {
    /* 裏面を示す暗い背景 */
    background: #336;
    filter: brightness(0.3);
}

/* 加槓の重ね表示 */
.kakan-stack {
    position: relative;
    display: inline-block;
}
.kakan-stack .tile-meld:nth-child(2) {
    position: absolute;
    top: -8px;
    left: 0;
}
```

**レイアウト修正:**
- `.melds-area` を手牌の右側に明確に分離
- 上下プレイヤー: `flex` で手牌→スペース→副露の順
- 左右プレイヤー: 副露を手牌の下に配置

### Step 3: applyActions() の副露処理確認

**確認項目:**
- 鳴いた時に手牌から正しい枚数が減っているか
- チーの場合: taken_tile以外の2枚が手牌から減る（taken_tileは河から来る）
- ポンの場合: taken_tileが手牌から2枚減る
- 河から鳴いた牌が正しく除去されているか

### Step 4: テスト

1. 鳴きが多い局のHTMLを生成（seed=67: ポン3、チー7、加槓1）
2. ブラウザで確認
3. 各鳴きの横向き牌位置が正しいか検証
4. 手牌枚数が正しいか（13 - 鳴き×3 + ツモ1 = 正しい枚数）
5. 河から鳴いた牌が消えているか確認
6. 全27テストが通ること
